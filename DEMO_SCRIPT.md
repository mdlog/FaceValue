# FaceValue — Demo Script (2–3 min video)

> **Stellar Hacks: Real-World ZK** submission. One sentence to open with:
> *"FaceValue stops ticket scalping with a zero-knowledge proof — the resale
> price stays private, but a price above the cap is mathematically impossible to
> prove, so the contract refuses it."*

This is a **shot list** mapped to the real running app. Times are targets, not
hard cuts. Run `npm run zk:build` once before recording so the proving artifacts
exist (see "Pre-flight" at the bottom); if you skip it, the app still runs — it
falls back to a clearly-labelled simulated verdict (disclosed below).

---

## What is REAL vs. disclosed-mock (say this on camera, ~10s)

Be explicit. Honesty is a judged criterion.

- **REAL** — The Circom circuit (`circuits/resale.circom`: `price ≤ cap` +
  Merkle membership + Poseidon nullifier), the **Groth16 proof** generated by
  snarkjs, and the **Groth16 verification** against `verification_key.json`.
  When you pick **Scalper $210**, the proof **genuinely fails to generate** —
  the witness is unsatisfiable because `210 > 120`. That failure is the
  enforcement; it is not a scripted `if`.
- **DISCLOSED-MOCK** —
  - Proving runs **server-side** (`/api/prove`) for a reliable demo. Production
    intent is **client-side** proving in the browser. State this.
  - The **Soroban contract** (`contracts/facevalue`) is written and
    reference-faithful to Stellar's `groth16_verifier`, but **testnet is not
    wired** in this build (the `stellar` CLI is absent in the demo
    environment). The on-chain verdict UI mirrors what the contract checks; the
    "Soroban · testnet" badge is illustrative.
  - Catalog data, addresses, USDC settlement, and the auditor view-key are
    **illustrative fixtures**, not a live anchor.

If `npm run zk:build` was **not** run, the resale page shows a small
**"simulated"** note and decides the verdict from `price ≤ cap` directly — say
"the artifacts aren't built in this recording, so this verdict is simulated"
rather than implying a real proof ran.

## Pre-roll (optional, 0:00–0:10)

Landing page `/` — the one-liner positioning: *regulated, private ticket
resale; enforcement is the headline, privacy is the wedge.* Quick pan over the
nav: **Resale · Door · Audit**.

## Beat 1 — The scalper is stopped by math (0:10–1:00)

**Open `/resale`.** (Event: *Aurora*, public cap **$120.00**.)

1. Point at the right panel — **"The proof will assert"** block:
   ```
   assert  resale_price ≤ per_event_cap
   assert  ticket ∈ merkle(issued_set)
   emit    nullifier = Poseidon(ticket_secret)
   ```
   Note `per_event_cap = $120.00` and `merkle_root` are **public**;
   `resale_price` and `buyer / seller` read **PRIVATE**.

2. Click **Scalper · $210**. The cap meter goes over the line.

3. Click **Generate proof & submit**. The stepper advances:
   *Sealing private inputs → Generating Groth16 proof → Verifying in Soroban*.

4. **Verdict: DENIED.** Land on the reject stamp. Say the line:
   > *"The proof can't be produced — `resale_price > cap` is unsatisfiable. No
   > ownership changed, no funds moved. The scalper is stopped by mathematics,
   > not policy."*

   (Real path: `/api/prove` returns `overCap:true` because the witness fails the
   `LessEqThan` constraint. Fallback path: simulated reject because `210 > 120`.)

## Beat 2 — A fair resale clears, privately (1:00–1:50)

5. Click **Fair · $115** (under the $120 cap). Cap meter back in the green.

6. Click **Generate proof & submit** again. The same stepper runs — this time a
   **real Groth16 proof is produced and verifies**.

7. **Verdict: ACCEPTED.** Show the **TxStamp** with the real tx-like hash
   (derived from the proof's `nullifierHash` public signal, truncated).
   Point out:
   - The old stub **`AUR-0482-7K` is nullified** — struck through.
   - A **fresh QR** is issued to the buyer.
   - Buyer, seller, and the exact $115 price **never appeared** — only the
     public signals `[per_event_cap, merkle_root, nullifier]` did.

   Say:
   > *"Same contract, same cap. Because $115 clears $120, the proof verifies,
   > the old ticket is burned by its nullifier, and the buyer gets a clean
   > ticket — without anyone learning who paid what."*

## Beat 3 — The nullified ticket is dead at the gate (1:50–2:20)

8. **Open `/door`.** Pick the **nullified / resold** ticket → **Scan**.
   Verdict: **Refused** (already resold — the old QR can never admit twice).

9. Pick an **active** ticket → **Scan** → **Admit**. One nullifier, one
   admission: the anti-double-sell guarantee, visible.

## Beat 4 — Disclosure on warrant, not by default (2:20–2:50)

10. **Open `/audit`.** The Exhibit is **redacted** — the public ledger reveals
    nothing.

11. Paste the scoped **view key** `vk_aur_9f31c0` → **Unseal**. Exhibit A
    unseals **one** record: buyer `GBUY…7Q2X`, seller `GSEL…3K9D`, exact price
    **$115.00**, tied to that nullifier.

    Say:
    > *"Privacy by default, disclosure on warrant. A regulator with a scoped
    > view key reconstructs a single transaction — not the whole book."*

## Close (2:50–3:00)

> *"One circuit enforces the cap, burns the old ticket, and keeps the price
> private — verified on Stellar. Remove the proof and you can't enforce a price
> ceiling without trusting a platform. That's why the ZK is load-bearing."*

## Pre-flight (do this before recording)

```bash
npm install
npm run zk:build      # compiles resale.circom, runs groth16 setup,
                      # emits public/circuits/{resale.wasm,resale_final.zkey,verification_key.json}
                      # and generates src/lib/zk/fixtures.json
npm run dev           # http://localhost:3000
```

Smoke-test before the take:
- `/resale` → **Scalper $210** → **DENIED** (and no "simulated" note shown).
- `/resale` → **Fair $115** → **ACCEPTED** with a real tx-like hash.
- `/door` nullified → Refused; active → Admit.
- `/audit` → `vk_aur_9f31c0` → unseals.

If you see a **"simulated"** note on `/resale`, the artifacts didn't build — fix
`zk:build` or disclose the simulation on camera.
